依赖倒置原则 避免用具体类，而多使用抽象。
而好莱坞原则 是用在创建框架或组件上的一种技巧
好让底层组件能够被挂钩进计算中，而且又不会让
高层组件依赖低层组件。二者的目标都是在与解耦
但依赖倒置原则更加注意如何在设计中避免依赖。

好莱坞原则是一个弹性的设计，允许低层结构能够
互相操作，而又防止其他类太过依赖他们

策略模式与模版方法的比较
二者侧重点不同
模版方法定义一个算法的大纲，在算法中的个别步骤
可以不同的实现细节，算法结构依然会维持不变。
（效率高 重复代码少，依赖超类中方法的实现）

策略模式不是继承进行算法的实现，通过对象组合的
方式，让客户可以选择算法的实现（因采用对象组合，
所以更有弹性，根据客户所需，改用不同的策略对象）

策略模式 封装可互换的行为，然后使用委托来决定要
采用哪一个行为
模版方法 子类决定如何实现算法中的某些步骤
工厂方法 由子类决定实例化个具体类

组合模式 ：有数个对象的集合，他们彼此之间有“整体
/部分”关系，想用一致的方法对待这些对象 P376 383
